心得

老師要求我們給自己與組員每週的評分及每週上完課的心得

第一週

開學的第一週的第一堂課就是嚴老師的協同產品設計實習,老樣子,前30分鐘跟我們分享一些時事與老師的想法及做事情的態度,接下來提及這一學期要做的事,大致上來說就是上學期的進階版,更為深度的探討。

第二週

這學期老師用隨機分組程式分好組,不再是那些最熟的同學,目的是為了讓我們脫離舒適圈,採取每周自評及組員互評,從各自做的東西來評分,這樣可以看出其他組員有沒有做出該做的東西來,希望我們這組可以合作愉快。

第三週

這週確定了組員名單,發現有三個學長,也許是剛開始不太熟,要找到學長好像有點困難,加上學長們太久沒上,有些東西已經忘了,需要稍微講解一下。

第四週

上課上到一半發現網路無法連線,結果是IP被占用,我們被踢出去,原本我們是使用ipv4,暫時切到ipv6,還是覺得不太穩定,推送時有點害怕出什麼差錯,會害網站爆掉。

第五週

分配實習任務一裡的取分項目,我是分配在項目三:配置ungit 為 ungit 寫一個使用手冊,先看了老師在blogger講解的就了解了一些,和組員討論後更加清楚ungit的操作。

第六週

本週開始製做實習任務二裡的BubbleRob 。

第七週

本週和小組分工製作實習任務二的手足球系統,我是負責繪圖的部分,使用onshape畫出手足球的場地,另一同學則是負責球員及竿子;照著老師給的尺寸做發現很多地方還是得自己設計尺寸出來,單位還是英吋還是有點不習慣。

第八週

上週做的原型在vrep組看過之後在本週需要修改一些死角的部分及製作一些細節使其更加完整。

Final

期末的任務跟期中的相去甚遠，後者較偏向單打獨鬥，一個小錯誤不至於影響很大，期末就不同了，大部分時間在設計與畫圖的我，在Onshape上看似沒問題，丟入Vrep模擬時一堆問題，又要回推去優化 改進，來來回回了不少次才解決問題，這完全凸顯了團隊溝通的重要性。這次我負責的領域是onshape分工及優化 和模擬部門溝通，接觸V-rep的時間沒有很長，希望以後能多多了解他，畢竟只是學了些皮毛，然後能寫一些厲害的程式出來。

編寫ungit使用手冊

1.下載 ungit 與 node.js 程式碼

首先下載 kmol_2019.7z, 接著再下載 nodejs_with_ungit.7z, 兩者分別解開壓縮檔案,然後將 nodejs 放入可攜 data 中

2.設定命令列搜尋路徑

接下來設定啟動 start.bat 中的 ungit 命列列搜尋路徑:

新增set path8=%Disk%:\nodejs;%Disk%:\nodejs\appdata\roaming\npm;(抓取nodejs主程式路徑)及

path=%path%;%path1%;%path2%;%path3%;%path4%;%path5%;%path6%;%path7%;%Disk%:\p37\site-packages\scipy\extra_dll;%Disk%:\scite\bin;%path8%;

3.設定 USERPROFILE 環境變數

讓 node.js 能夠使用 y:\home 作為使用者目錄, 在 start.bat 中加入

set USERPROFILE=%Disk%:\home

4.在 SciTE 編輯器輸入指令, 指令如下 :

ungit --launchBrowser=0 --launchCommand "C:\PROGRA~2\Google\Chrome\Application\chrome.exe  %%U"

完成存在y:\下 (跟ungit做一個區別批次檔名稱改成kungit)

5.開啟start.bat,輸入kungit後,會在瀏覽器中開啟ungit頁面

〈組合圖〉

這是最一開始的初步組合圖，

場地四周內側設計有圓角、場地上有突出障礙物以及回球機構

設計理念:

1.圓角:以防球模擬時跑到角落就不會動了

2.障礙物:多點障礙物，增加遊戲難度

3.回球機構:任何一方得分後，球會自動回到場上

但由於在模擬及簡化時有問題

就砍掉之前的場地設計及距離

因為過多增加系統負擔的圖形，只會拖慢模擬速度

而回球機構我們設計了一個擊球系統

擊球系統:任何一方得分後，球進入軌道，會由擊球機構把球擊出並沿著軌道回到被得分那一方上
